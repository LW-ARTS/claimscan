# Phase 12.1 Plan 03: Type-Surface Widening + SplitVault ABI

Widened the `FlapVaultKind` union and added the `SPLITVAULT_USERBALANCES_ABI` constant so the upcoming handler (Plan 04) and probe ladder (Plan 05) have type-safe entry points.

## What Was Built

Three files modified atomically:

1. **`lib/platforms/flap-vaults/types.ts`** — Added `'split-vault'` to `FlapVaultKind` union and exported `SPLITVAULT_USERBALANCES_ABI = parseAbi(['function userBalances(address user) view returns (uint128 accumulated, uint128 claimed)'])` next to the existing `V2_PROBE_ABI`.
2. **`lib/platforms/types.ts`** — Widened `TokenFee.vaultType?` literal union to include `'split-vault'`.
3. **`lib/supabase/types.ts`** — Widened the `flap_tokens.vault_type` literal union in 3 places (Row, Insert, Update tables). Did NOT touch `fee_records.vault_type` (stays `string | null`).
