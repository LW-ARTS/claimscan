# Phase 12.1 Plan 07: Integration Parity Test (Find-One-At-Runtime)

Drove Plan 01's it.todo integration stub to a functional state using the find-one-at-runtime pattern, with descriptive skipIf messages that prevent silent false-positive verification.

## What Was Built

`lib/__tests__/integration/flap.test.ts` extended with:
- New imports: `createServiceClient`, `SPLITVAULT_USERBALANCES_ABI`, `bscClient`
- Test body queries `flap_tokens` for `vault_type='split-vault' LIMIT 50`
- Loops candidates calling `userBalances(creator)` directly via bscClient
- First candidate with `accumulated > claimed`: calls `flapAdapter.getHistoricalFees(creator)` and asserts the matching row's `totalUnclaimed === (accumulated - claimed).toString()`
- If no candidates in DB → `it.skip` with message "no vault_type=split-vault candidates in DB, skipping"
- If all 50 candidates have 0 claimable → `it.skip` with message "0/N had non-zero claimable, skipping (steady-state)"

Two distinct skip paths matter: a uniform "skipped" status would mask both "phase data not migrated" (real bug) and "expected steady state" (correct behavior).

## Expected Runtime Outcomes

- Without `BSC_RPC_URL` → outer describe.skipIf triggers, test skipped uniformly with the existing message
- With `BSC_RPC_URL` set, no Supabase env → throws inside `createServiceClient()` (5th outcome path, documented in deviation log)
- With both envs, no split-vault rows → `it.skip` with message A
- With both envs, all rows zero-claimable → `it.skip` with message B (steady-state expected)
- With both envs and at least one non-zero → asserts adapter parity (PASS)
